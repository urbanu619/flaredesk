# Flaredesk 完全新手安装指南（Windows / macOS）

面向：**完全零基础**（平时不写代码、不熟悉命令行也可以跟着做）。  
本文尽量用**自然语言**说明：每一步要做什么、做完**正常情况**下你会看到什么。若某一步和下面写的不一样，把**完整报错原文**复制下来再问人或在社区提问，会更容易解决。

---

## 先选一条路：你要哪种用法？

| 你想要的效果 | 难度 | 说明 |
|-------------|------|------|
| **只在 AI 里（如 Cursor / Claude）用自然语言管理 Cloudflare DNS** | ⭐ 最低 | **不用**安装本仓库、不用数据库。只需装 Node，再配一条 MCP。 |
| **用浏览器打开网页面板**（多账号、批量 DNS 等） | ⭐⭐⭐ 较高 | 需要在本机安装 **Go、Node**；数据库可用 **SQLite**（免装 MySQL）、Redis 可用 **内嵌模式**（免装 Redis 服务）。若坚持用 MySQL + 独立 Redis，步骤更多。 |

**建议：** 若你是第一次接触这类工具，可先完成 **MCP 这一条路**，确认 Cloudflare Token 能用，再决定是否折腾完整面板。

下面分两部分写：**A. 最简单（MCP）**、**B. 完整网页面板（当前仓库方式）**。

---

## 名词速查（看不懂就回来看）

- **终端 / 命令行**：一个黑色或白色窗口，里面可以输入英文命令。Windows 上常用 **PowerShell** 或 **cmd**；macOS 上叫 **终端（Terminal）**。
- **路径 / 文件夹**：就是你平时在资源管理器 / Finder 里点的目录。文档里写的 `cd 某个目录` 表示「先进入这个文件夹再执行后面的命令」。
- **粘贴**：在终端里一般用 **右键** 粘贴（Windows），或 **Command + V**（macOS 终端默认支持）。

## A. 最简单：只用 MCP（不部署 Flaredesk 网页）

**你会得到：** 在支持的 AI 工具里用中文说「列出我的域名」「给某某加一条 A 记录」等。

### A1. 安装 Node.js（两种系统都要）

1. 打开官网：https://nodejs.org/  
2. 下载 **LTS（长期支持版）**，一路下一步安装。  
3. **验证是否装好：**

**Windows：** 打开 **PowerShell**（开始菜单搜索 `PowerShell`），输入：

```text
node -v
```

按回车。  
**正常情况：** 会显示类似 `v20.x.x` 的一行版本号。

**macOS：** 打开 **终端**（Spotlight 搜索 `终端` 或 `Terminal`），同样输入 `node -v`。  
**正常情况：** 同上。

若提示「不是内部或外部命令」或 `command not found`，说明 Node 未装好或未加入路径，请关闭终端重新打开再试，或重新安装 Node。

### A2. 在 AI 工具里配置 MCP

具体配置步骤见仓库根目录 [README.md](../README.md) 里的 **「MCP Server（AI 工具集成）」** 一节。  
你需要准备：**Cloudflare API Token**（权限至少包含对 Zone DNS 的编辑能力）。

**正常情况：** 配置保存并重启 AI 客户端后，在对话里让它执行 `list_zones` 一类操作，能返回你的域名列表。

## B. 完整网页面板（本机运行前后端）

**你会得到：** 浏览器里登录 Flaredesk，使用多账号、批量 DNS 等全部功能。  
**你需要接受：** 步骤较多；本机至少要装 **Go、Node**。数据库与缓存可选 **SQLite + 内嵌 Redis**（见下），不必再装 MySQL / Redis 服务。

### B1. 把代码放到本机

**方式一（推荐会一点命令行的用户）：** 安装 Git 后克隆仓库。  
**方式二（完全不想装 Git）：** 在浏览器打开 GitHub 项目页，点绿色 **Code → Download ZIP**，解压到例如桌面上的 `flaredesk` 文件夹。

**正常情况：** 你能在文件夹里看到 `magic_admin`、`magic_admin_web` 等目录。

### B1a.（推荐）单机极简：SQLite + 内嵌 Redis

**你会跳过：** 单独安装 MySQL、单独安装并启动 Redis、手动建库（`CREATE DATABASE`）。

1. 在资源管理器 / Finder 中打开：`magic_admin/conf.d/`。  
2. 将仓库里的示例文件 **`config.sqlite.embedded.json.example`** **复制一份**，改名为 **`config.json`**（若已有 `config.json`，可先备份再覆盖，或把示例里的字段合并进现有文件）。  
3. 确认 JSON 里 **`mysql` 段有 `"driver": "sqlite"`**，**`path`** 为数据目录（如 `"./data"`），**`db-name`** 会用来生成 `./data/库名.db` 文件；**`redis` 段有 `"embedded": true`**。

**正常情况：** `magic_admin/conf.d/` 下存在可用的 `config.json`，且 `jwt.signing-key` 已改成你自己的随机字符串（与 README 说明一致）。

完成 **B1a** 后，请直接跳到 **B5 启动后端**（跳过下面的 **B2、B3**，**B4** 若已用示例配置也可略过，只需检查 `jwt`）。

---

### B2. 安装运行环境（仅在使用 MySQL + 独立 Redis 时需要）

下面每个工具装好后，**都建议关掉再重新打开终端**，再执行验证命令。

| 软件 | 用途 | 官网 / 说明 |
|------|------|-------------|
| **Go** | 运行后端 | https://go.dev/dl/ |
| **Node.js** | 构建前端 | https://nodejs.org/ （LTS） |
| **MySQL** | 数据库 | Windows 可装 MySQL Installer；macOS 可用官网包或 Homebrew |
| **Redis** | 缓存等 | Windows 可用官方或 WSL；macOS `brew install redis` 等 |

**验证命令（在终端执行）：**

```text
go version
node -v
mysql --version
redis-cli ping
```

**正常情况：**  
- `go version` 显示 `go1.21` 或更高。  
- `node -v` 显示版本号。  
- MySQL、Redis 能连上（`redis-cli ping` 返回 `PONG`）。

若某一条报错，先解决该软件安装，再继续。

> **走 B1a（SQLite）的用户：** 不要执行 B2、B3。

### B3. 创建数据库（仅 MySQL）

用 MySQL 客户端（或命令行）执行：

```sql
CREATE DATABASE flaredesk DEFAULT CHARACTER SET utf8mb4;
```

**正常情况：** 执行成功，无报错。

> **走 B1a（SQLite）的用户：** 不要执行本节建库 SQL。

### B4. 改配置文件

用记事本（Windows）或文本编辑（macOS）打开：

`magic_admin/conf.d/config.json`

- **若已用 B1a 示例：** 主要检查 **`jwt.signing-key`** 是否已改为随机字符串。  
- **若使用 MySQL：** 按 [README.md](../README.md) **「部署步骤 → 配置后端」** 填写 `mysql`、`redis`（或设 `redis.embedded: true` 免独立 Redis）、`jwt`；密码填本机 MySQL 密码。

**正常情况：** 文件能保存；JSON 格式正确（逗号、引号成对，最后一项后面不要多余逗号）。

### B5. 启动后端

**Windows（PowerShell）：**

```powershell
cd 你解压的目录\flaredesk\magic_admin
go run main.go api
```

**macOS（终端）：**

```bash
cd ~/你的路径/flaredesk/magic_admin
go run main.go api
```

**正常情况：** 终端里持续滚动日志，没有立刻退出；最后一两行可能显示监听端口 **2022**。  
**若报错：** 把从 `go run` 开始到报错结束的**全部文字**复制保存。

首次成功启动会自动建表并创建默认管理员账号（见 README）。

### B6. 构建前端

**新开一个终端窗口**（保留后端窗口不要关），执行：

**Windows：**

```powershell
cd 你解压的目录\flaredesk\magic_admin_web
npm install
npm run build
```

**macOS：**

```bash
cd ~/你的路径/flaredesk/magic_admin_web
npm install
npm run build
```

**正常情况：** `npm run build` 结束时有 `build` 成功提示；生成 `magic_admin_web/dist` 文件夹。

### B7. 用浏览器访问（新手最简单：本机临时起一个静态服务）

若没有装 Nginx，可以先用 Node 自带的方式**临时**打开前端（仅本机测试）：

```bash
cd magic_admin_web/dist
npx --yes serve -s . -l 3000
```

然后在浏览器地址栏输入：`http://127.0.0.1:3000`

**注意：** 前端需要能访问后端 API。默认前端会请求 `/admin/api/` 一类地址，你需要按 README 用 **Nginx** 或等价方式把 `/admin/api/` 代理到 `http://127.0.0.1:2022`，否则登录接口会失败。**若完全不懂代理**，这是新手最容易卡住的地方——建议查阅 README 里的 Nginx 示例，或请熟悉服务器配置的人协助一次。

**正常情况：** 页面能打开；若代理已配对，可用默认账号登录（登录后请立即改密码）。

## 遇到问题怎么描述，别人更好帮你

1. **你的系统：** Windows 11 还是 macOS 哪一版。  
2. **你走的是 A（MCP）还是 B（完整面板）。**  
3. **你执行的最后一条命令**（可复制粘贴）。  
4. **终端里完整报错**（从命令到最后一行错误）。  
5. **你预期发生什么、实际发生了什么**（一句话）。

## 和「一键部署」的关系

完全小白最省力的是：**官方提供带界面的一键安装包**（或单 exe / 单 app），内置数据库、自动打开浏览器——这需要产品开发。当前仓库已支持 **SQLite + 内嵌 Redis**，可少装两个外部服务；若再提供预编译二进制，步骤还可再缩短。

在包出现之前，本文把步骤拆成「先 MCP、再面板」，并写明**正常输出**，是为了减少「猜一步错一步」的挫败感。若你愿意把踩坑过程写成图文教程，也会对其他零基础用户帮助很大。
